# 画面構成設計

## ボトムナビゲーション

アプリのメイン画面は 4 つのタブで構成されます。

```
┌─────────┬─────────┬─────────┬──────────┐
│  発見   │ マップ  │ しおり  │アカウント│
│  🔍    │  🗺️   │  🔖   │   👤    │
└─────────┴─────────┴─────────┴──────────┘
```

### 日英対応表

| 日本語     | 英語    | アイコン | 説明                             |
| ---------- | ------- | -------- | -------------------------------- |
| 発見       | Explore | 🔍       | 新しい聖地を探す                 |
| マップ     | Map     | 🗺️      | 地図上で聖地を確認する           |
| しおり     | Saved   | 🔖       | お気に入りの聖地（行きたいリスト） |
| アカウント | Account | 👤       | ユーザー情報と過去の記録         |

---

## 各タブの詳細

### 1. 発見（Explore）

**目的**: 新しい聖地を見つける

**主な機能**:

- 作品検索
  - キーワード検索（作品名）
  - 検索履歴
- 人気作品一覧
  - 登録聖地数の多い作品
  - 最近人気の作品
- おすすめ聖地
  - 新着聖地
  - 話題の聖地

**ユーザージャーニー**: 事前計画フェーズ（家で使う）

---

### 2. マップ（Map）

**目的**: 地図上で聖地を確認する

**主な機能**:

- 地図表示
  - 聖地のピン表示
  - 詳細情報のポップアップ
- 現在地周辺の聖地表示
  - 位置情報を利用した近隣検索
  - 距離の表示
- フィルター機能
  - 作品別フィルター
  - 距離別フィルター

**ユーザージャーニー**: 移動中・現地フェーズ（実際に巡礼する）

---

### 3. しおり（Saved）

**目的**: 行きたい聖地を管理する（未来志向）

**主な機能**:

- お気に入り聖地一覧
  - ユーザーが「行きたい」に追加した聖地
  - リスト表示 / 地図表示の切替
- 並び替え・フィルター
  - 作品別
  - 地域別
  - 追加日順

**ユーザージャーニー**: 事前計画フェーズ（行きたい場所をリストアップ）

**注**: 過去の記録（訪問済み、レビュー）は「アカウント」タブに配置

---

### 4. アカウント（Account）

**目的**: ユーザー情報と過去の記録を管理する

**主な機能**:

- プロフィール
  - ユーザー名、アイコン
  - 訪問数、レビュー数、投稿数の統計
- 訪問済みリスト
  - 実際に訪れた聖地の履歴
  - 訪問日の記録
- 自分のレビュー
  - 投稿したレビュー一覧
  - 評価、コメント、写真
- 投稿管理
  - 投稿した聖地の一覧・ステータス確認
  - リクエストした作品の一覧・ステータス確認
- 設定
  - 多言語切り替え（日本語 / 英語）
  - アカウント情報の編集
  - 退会機能

**ユーザージャーニー**: 振り返りフェーズ（帰宅後に記録を見返す）+ 全フェーズ（設定）

---

## 設計の考え方

### 未来と過去の棲み分け

- **しおりタブ（未来）**: これから行きたい場所
- **アカウントタブ（過去）**: すでに行った場所、レビュー

この明確な棲み分けにより、ユーザーが迷わず目的の情報にアクセスできます。

### ユーザージャーニーとの対応

1. **事前計画**: 発見 / しおり
2. **移動中・現地**: マップ
3. **振り返り**: アカウント

各フェーズに対応したタブ構成により、ユーザーの行動フローに沿った自然な導線を実現します。

## 今後の拡張

将来的に以下の機能を追加する可能性があります：

- 聖地巡礼ルートの作成・共有
- ユーザー同士のフォロー機能
- コミュニティ機能（掲示板、イベント情報）

その際は、タブ構成の見直しや、新しいセクションの追加を検討します。
