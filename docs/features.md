# アニメ聖地巡礼アプリ 機能要件一覧

## 1. ユーザー側機能（モバイルアプリ）

### A. アカウント・設定

- **会員登録 / ログイン**
  - ソーシャルログイン（Google, Apple 等）を優先
- **多言語切り替え**
  - 日本語 / 英語（UI の言語切替）
- **マイページ**
  - ユーザー情報の確認・編集
  - 退会機能
  - 自分が投稿した聖地の一覧・ステータス確認
  - 自分がリクエストした作品の一覧・ステータス確認

### B. 検索・閲覧（コア機能）

- **作品検索**
  - アニメタイトルによるキーワード検索
- **聖地一覧表示**
  - 検索結果のリスト表示
- **聖地マップ表示**
  - Google Maps 等を利用したピン表示
  - 現在地周辺の聖地表示（推奨機能）
- **聖地詳細画面**
  - スポット写真（現地の風景）
  - 住所 / アクセス情報
  - アニメのシーン・登場背景のテキスト説明

### C. アクション・記録

- **レビュー投稿**
  - 評価（星 5 段階など）
  - コメント入力
  - 写真アップロード機能（現地写真の共有）
- **訪問記録（行ったよリスト）**
  - 訪れた聖地の履歴管理
- **聖地登録機能（ユーザー投稿）**
  - 既存作品を選択
  - スポット名、住所 / 位置情報、説明文、写真を入力
  - 管理者承認後に公開
  - 投稿ステータスの確認（承認待ち / 承認済み / 却下）
- **作品リクエスト機能**
  - 未登録の作品を追加リクエスト
  - 作品名（日・英）と簡単な説明を入力
  - リクエストステータスの確認

---

## 2. 管理者側機能（管理画面）

### A. マスターデータ管理

- **作品（タイトル）管理**
  - 新規追加・編集・削除
  - 日英タイトルの登録
- **聖地スポット管理**
  - スポット情報の登録・編集・削除
  - 緯度・経度情報の登録（マップ用）
  - スポット説明文の登録（日・英）

### B. コンテンツ・ユーザー管理

- **レビュー管理**
  - 投稿一覧の確認
  - 不適切コメント・画像の非表示/削除対応
- **聖地投稿管理**
  - ユーザー投稿された聖地の一覧表示
  - 投稿内容の確認（位置情報、写真、説明文）
  - 承認 / 編集 / 却下のアクション
  - 却下時の理由記載
- **作品リクエスト管理**
  - リクエスト一覧の確認
  - リクエスト詳細の確認
  - 承認（作品マスターに追加）/ 却下 / 保留のアクション
- **ユーザー管理**
  - ユーザー一覧
  - アカウント停止（BAN）処理
  - 投稿数の多いユーザーの確認（貢献度管理）

### C. システム運用

- **お知らせ管理**
  - アプリ向け通知（新着聖地情報など）の作成・配信

---

## 3. 非機能要件・考慮事項

- **多言語対応:** コンテンツ（聖地データ）の日英併記運用フローの確立
- **著作権対応:** アニメ本編のキャプチャ画像を使用せず、テキストや現地写真で構成する設計
- **拡張性:** 将来的な Android/iOS 両対応（Cross-platform 開発の検討）
